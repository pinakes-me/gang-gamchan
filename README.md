# 강감찬자료실 LOD 파일럿 (gang-gamchan-lod)

관악구립도서관 강감찬자료실의 텍스트 목록을 Linked Open Data로 변환하는 파일럿.
연구 프레임: 공공도서관 향토자료실의 LOD 적용 가능성 연구 — 관악구 강감찬자료실 사례.

## 구조

```
data/books.csv        정제된 도서목록 (현재 1~2페이지, 19건)
vocab/concepts.csv    통제어휘 초안 (SKOS, 26개 개념 + 9개 상위범주)
schema/ontology.ttl   데이터 모델 스케치 (schema.org + DC Terms + SKOS + 최소 커스텀)
scripts/csv_to_ttl.py CSV → Turtle 변환 파이프라인
output/               생성된 concepts.ttl, books.ttl
index.html             탐험 지도 프로토타입 (단일 파일)
```

## 파이프라인

1. **수집**: 도서관 사이트에서 페이지별 표 복사 → `data/books.csv`에 행 추가
2. **태깅**: `subjects` 컬럼에 `vocab/concepts.csv`의 concept_id를 세미콜론으로 부여.
   새 개념이 필요하면 어휘에 먼저 추가 (전거제어 원칙: 어휘가 데이터에 선행)
3. **변환**: `python scripts/csv_to_ttl.py`
4. **검증**: `review_flag=Y` 행은 원자료 확인 후 해소

### 방법론 정정 (2026-07-06)
초기 분석에서 발행년도 분포를 '수집 이력'으로 해석했으나, 이 목록은 강감찬
브랜드화 사업의 일환으로 **한 시점에 일괄 구축되었을 가능성**이 크다(Hailey 지적).
이 경우 발행년 분포는 입수 행위가 아니라 출판 시장 동향의 반영이며,
"수집정책 없이 성장한 컬렉션" 같은 서술은 근거 없음. 논문에서는
'일괄 구축 가설'과 '점진 수집 가설'을 병기하고, 구축 시점은 도서관 확인 필요
사항으로 남길 것.

### 소장처 위치 데이터 정정 (2026-07-06)
'Google Places·서울도서관 디렉토리 미등재'로 기록했던 작은도서관들은 폐관이 아니라
**동주민센터 작은도서관**(자치행정 소속)이어서 도서관 디렉토리에 없었던 것.
관악구 동주민센터 작은도서관 현황(Hailey 확보)으로 9곳 좌표 보완 완료.
→ 관찰 수정: "작은도서관의 데이터 비가시성"은 폐관·미등록이 아니라
**소관 부서에 따른 데이터 분절**(도서관 시스템 vs 자치행정 시스템)의 문제.
꿈나무영유아도서관만 위치 확인 중.

### 참고목록 갱신 지연 확인 (2026-07-06, Hailey 직접 확인)
181번 『고려거란전기』(길승수, 2018) — 원천 목록에는 소장 '-'로 표기되어 있으나,
관악구통합도서관 실제 검색 결과 **관악중앙도서관·글빛정보도서관에 소장 확인됨**.
드라마(2023~24) 원작 소설임에도 참고목록 페이지의 소장 여부 컬럼이 갱신되지 않은 것.

→ "목록이 2020년에 멈췄다"는 기존 발견의 정밀화: 실제 소장(수서)은 계속 진행되고
있으나, 강감찬자료실 참고목록 페이지가 그 갱신을 반영하지 못하고 있음.
정적 참고목록과 살아있는 소장 현황 간의 구조적 분리 — 아마도 목록 관리와
참고서비스 페이지 관리가 별도 업무라 후자가 방치되는 경우로 추정.

**CSV는 수정하지 않음** — 이 사이트/데이터셋의 연구 대상은 '현재 실제 소장 현황'이
아니라 '원천 목록이 특정 시점에 무엇을 등재했는가'이므로, 원천 범위를 벗어난
업데이트는 연구 대상 자체를 훼손함. 이 불일치는 발견으로 기록만 하고,
목록 갱신은 도서관 측에 게시판으로 요청.
